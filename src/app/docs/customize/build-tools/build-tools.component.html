GlobalUI comes with sass modules for all the components.
  • How it works:  Build tool takes the customization information from the json file and compiles the sass and finally generate the css file to dist folder

  • It Uses:  It uses the gulp (node framework) framework to build the tasks.

    Customization with CLI requires a custom.json file which is available in the customize folder of the root directory.
    Structure of custom.json is given below
    { "vars": { // <-- variables "colors": [ ], // <-- site primary colors palette "site": { // <-- others site vars "colors": { }, // <-- Generic site colors "font": { }, // <-- site fonts vars "dimensions": { } // <-- dimensions of components } }, "modules": { // <-- what to include "extra_colors": [ ], // <-- selection of fixed color palette "components": [ ] // <-- selection of sass modules } }
    Options:
  • vars :   contains the values of site variables
  • colors (array) :   Array of primary colors palette ( refer here for adding and removing color )
  • site :   Specifies site variables.
  • colors :   Specifies some Generic site colors
  • body-background (string) :   Background color of the body
  • body-text (string) :   Site text color.
  • link-color (string) :   color of the hyperlink.
  • font :   Specifies site font vars.
  • font-family (string) :   Site font family
  • dimensions :   Specifies dimensions of site components.
  • nav-height (string) :   Height of nav bar (top-bar) .
  • side-nav-width (string) :   Width of the Side drawer.
  • modules :   Specifies what to import ( select only what you need ).
  • extra_colors (array) :   Array of fixed color palette that you want to include in your site ( refer here for adding and removing color )
  • components (array) :   Array of site components you want to include to your site.
    • Default / Reset:
  • If the fields are not specified in the custom.json, then it will take the default values of GlobalUI vars.
  • Example 1:
    { "vars": { "colors": [ {"primary": "rgb(4, 131, 169)"}, {"secondary": "hsl(217, 33%, 31%)"} {"info": "hsl(217, 71%, 53%)"}, {"success": "hsl(141, 71%, 48%)"}, {"warning": "hsl(48, 100%, 67%)"}, {"danger": "hsl(348, 100%, 61%)"}, {"light": "hsl(0, 0%, 96%)"}, {"dark": "hsl(0, 0%, 21%)"} ], "site": { "colors": { "body-background": "#f6f6f6", "body-text": "#000", "link-color": "#3b5998" } } }, "modules": { "extra_colors": [ ], "components": [ 'button', 'grids' ] } }

    In above example,
  • build-tool will take the 6 primary colors specified in the colors field of the vars.
  • build-tool will take the generic site colors specified in the colors field of the site.
  • But build-tool will take the default font and dimensions variables since it is not mentioned in the file.
  • build-tool will not compile any extra-colors from the color-palette since extra_colors field is empty array.
  • build-tool will import only button & grids module to the final css file.

    • Points to remember,
  • If the build-tool want to compile default values then don't mention it.
  • So that, By removing the extra_colors field from the file will compile all the extra-colors from the palette.

    Once your custom.json file is ready, You are now ready to build.
    Steps:
  • Go to root directory.
  • Hit below command to build.
      gulp build
  • Now your CLI tool will compile all the sass files by taking the vars from the custom.json file.
  • After the successfull build, global-ui.css & global-ui.min.css file will generated in the dist folder.
