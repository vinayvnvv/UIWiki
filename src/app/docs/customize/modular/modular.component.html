GlobalUI comes with sass modules for all the components.
  • Works with only sass Integration:  To use modular feature , You must need sass compiler or sass integration to compile the the sass modules.
  • Angular Advantage:  If your application is angular 2 and above and having angular CLI, then by default it comes with sass integration.
  • Just Import:  If your application having either of the above two requirements, then you are ready to import the GlobalUI sass modules.

    ^v1.0
    You can import the modules by including that to your main sass file
    Example:
    @import 'global-ui/sass/include.sass' @import 'global-ui/sass/grids/grids.sass' @import 'global-ui/sass/elements/button.sass'
    In above example , you can access only grids and buttons modules so that it will not compile all the modules.
  • global-ui/sass/include.sass file contains basic styles & variables for your application to compile other modules.
  • You must include include.sass file to compile other modules.
    • {{toc_list[3].name}}
    ^v1.0
    Angular CLI has default sass compiler so that it's easy to import the variables and modules of GlobalUI.
    Steps:
  • Create main.sass file in src directory of your angular application.
      - include what you need in main.sass
      @import '~global-ui/sass/include.sass' @import '~global-ui/sass/grids/grids.sass' @import '~global-ui/sass/elements/button.sass'
      ~ symbol at start of the src in angular CLI automatically point to node_modules directory. So no need to include full path in @import link.
  • Link the main.sass file to angular application by specifying it in .angular-cli.json at styles field.
      Example:
      { "apps": [ { "styles": ["main.sass"] } ] }
  • That's all. Now your angular application importing only grids and button modules from GlobalUI bundle.
